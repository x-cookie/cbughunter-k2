# The 7-Question Gate

Every finding must pass all 7 questions before a report gets written.
**One wrong answer = kill the finding and move on.**

## The 7 Questions

1. **Is this in scope?**
   Check the program's scope page. Asset, subdomain, and vulnerability type must all be in scope. If unsure — it's out.

2. **Can I prove it's exploitable?**
   You need a working PoC. "Theoretically exploitable" is not exploitable. Reproduce it yourself.

3. **Is there real impact?**
   What does an attacker actually gain? Data exposure, account takeover, privilege escalation, financial damage. If the impact is "information disclosure of non-sensitive data" — reconsider.

4. **Did I reproduce it twice?**
   One reproduction could be noise. Two clean reproductions on the same path means it's real.

5. **Is the PoC clean?**
   Your PoC must be minimal, documented, and reproduce in isolation. No dependencies on your local state. Another researcher must be able to follow your steps and get the same result.

6. **Have I checked for duplicates?**
   Search HackerOne / Bugcrowd / the program's public disclosed reports. If a variant of this was reported and triaged as N/A, understand why before submitting.

7. **Would I be proud to send this?**
   If you'd hesitate to show this to a senior security engineer, it's not ready. Polish the impact statement, clean the title, verify the CVSS.

## The kill rule

If you answer **No** or **Unsure** to any question — **kill the finding**. Do not submit speculative reports. Your N/A ratio is your reputation.

## CVSS 3.1 quick reference

| Severity | Score | Common examples |
|---|---|---|
| Critical | 9.0–10.0 | Pre-auth RCE, SQLi on user PII, ATO at scale |
| High | 7.0–8.9 | Auth bypass, stored XSS on admin panel, SSRF to IMDS |
| Medium | 4.0–6.9 | IDOR (limited data), CSRF with low-value action, open redirect |
| Low | 0.1–3.9 | Info disclosure (non-sensitive), self-XSS |
| Informational | — | Findings with no direct exploitability |

## 60-second pre-submit checklist

```
[ ] Title uses: [VulnType] in [Component] allows [Impact]
[ ] Severity matches CVSS score
[ ] Steps to reproduce are numbered and complete
[ ] PoC includes request + response (or video)
[ ] Impact statement is concrete (what data, whose account, what privilege)
[ ] No duplicate found in program history
[ ] All 7 Gate questions answered Yes
```
